# Contributing to Opsly SaaS

First off, thank you for considering contributing to Opsly! It's people like you that make Opsly such a great tool for early-stage teams.

## 1. Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](../../issues) first. If it doesn't exist, feel free to open one!

## 2. Setting up your environment

1. Fork the repo and clone it locally.
2. Ensure you have Node.js installed.
3. Run `npm install` to install dependencies.
4. Set up your `.env.local` with Supabase and Stripe keys (see README).
5. Run `npm run dev` to start the local development server.

## 3. Pull Request Process

1. Create a new branch off `main` for your feature or bug fix: `git checkout -b feature/your-feature-name`
2. Make your changes and commit them with descriptive commit messages.
3. Push to your fork and submit a Pull Request.
4. Answer the questions in the PR template and link any relevant issues.
5. A maintainer will review your code. We may ask for some changes before merging.

## 4. Code Style

We use ESLint and Prettier. Please make sure your code passes linting before submitting a PR.
Run `npm run lint` to check for issues.

Happy coding!
